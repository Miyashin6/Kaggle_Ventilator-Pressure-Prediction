# `sample.ipynb` とは何か（初心者向け）

このドキュメントは、Kaggle の Ventilator Pressure Prediction コンペ向けノートブック `sample.ipynb` を、**機械学習の前提が少ない人**にも伝わるように説明したものです。

---

## 1. まず知っておくこと：このノートは「学習用」ではない

よくあるコンペのノートブックは次の流れです。

1. `train.csv` でモデル（ニューラルネットなど）を**学習**する  
2. `test.csv` で**予測**する  
3. `submission.csv` を提出する  

一方、`sample.ipynb` の役割は **③のあと、もしくは別のモデルが出した予測を「後から賢く直す」** ことに近いです。

| よくあるノート                     | `sample.ipynb`                       |
| --------------------------- | ------------------------------------ |
| データからゼロで予測を作る               | **すでにある予測（主にニューラルネットの提出ファイル）を入力にする** |
| LightGBM や NN の**学習コード**が中心 | **物理・制御の考え方（PID）** に基づいて数値を書き換える     |

つまり、**「第2段階の仕上げ用」**のノートブックです。タイトルにある *inverting the function of PID controllers* は、「制御器（PID）の式を**逆に解いて**、圧力の値を求め直す」という意味です。

---

## 2. コンペで何をしているか（超短く）

- 人工呼吸器が **気道圧（pressure）** をどう変えるかを、**制御入力**（`u_in` や `u_out`）から予測する問題です。
- 実機のシミュレーションでは、**PID 制御**のような仕組みで圧力が決まることがあります。
- このノートの作者は、「データの裏側は **P 制御だけ** か **PI 制御** に近い振る舞いをしている呼吸が多いのではないか」と考え、**その式に当てはまる部分の圧力は、ニューラルネットの予測より**制御の式から**逆算した方が正確**になることがある、としています。

---

## 3. 用語をやさしく

### P 制御 / PI 制御（ざっくり）

- **制御**＝「どれだけバルブを開くか（`u_in` など）」を決めて、「目標の圧力」に近づけること。
- **P（比例）**＝今の誤差に比例して操作量を変える、というシンプルなルール。
- **PI（比例＋積分）**＝P に加えて、過去の誤差の積み重ねも考える、という**少しリッチな**ルール。

このノートは「**本当の呼吸はこのルールにかなり従っている**」と仮定し、**`u_in` の時系列から、理論上の圧力を逆算**します。

### 「逆関数（inverse）」とは

- **順方向**：圧力（状態）が分かっている → 制御の式で「次の操作」や「次の圧力」を計算する。  
- **逆方向**：**操作（`u_in`）が分かっている** → 式を **圧力について解く** ＝ **逆関数**。

このノートは「**ニューラルネットの予測より、制御の逆算の方が正しい**」と判断できるタイミングで、**`pressure` を置き換える**、という処理です。

---

## 4. データの準備で何をしているか

ノートの前半では、大きく次のことをしています。

1. **`train.csv` から**  
   - 圧力がセンサー上 **どのくらいの刻み（離散値）** で取られるか（`p_min`, `p_step`）を推定する。  
   - 探索を速くするため、**よく出る圧力の値の順**なども作る。

2. **`test.csv` から**  
   - 1 呼吸あたり **80 ステップ**なので、行を `（呼吸の本数）× 80` の **2次元配列**に並べ直す。  
   - **`u_in`** → 配列 `uu`  
   - **吸気中かどうか**（`u_out == 0` の行だけが吸気の対象）→ 配列 `rr`  
   - **時間の差**（`dt_`）→ 制御の式で使う。

ここまでが「後処理のための下準備」です。

---

## 5. 中盤の処理の流れ（イメージ）

1. **`find_pi_control`**  
   - 各行（各呼吸）について、「この呼吸は **PI 制御**に合うか？」を試す。  
   - 合うと判断できたら、**係数（`p_coef`, `i_coef`, 目標圧 `p_star` など）** を推定し、**その式から圧力を再計算**して、元の予測を更新する。

2. **`find_p_control`**  
   - PI だけでは説明しきれないものの、「**P 制御だけ**で説明できる」呼吸について、同様に**係数を推定**して圧力を更新する。

3. **学習データがあるとき**（変数 `pp` が定義されているとき）  
   - 更新で誤差が減るか**検証**する、というブロックもある（テストだけ実行すると `NameError` を捕捉して警告が出る、という動き）。

※ コードは非常に長く、**グリッド探索**（候補の係数を数多く試す）が入っています。細部は「コンペ上級者向けの実装」と読んで問題ありません。

---

## 6. 終盤：提出ファイルの読み込みと並列処理

1. **別の提出 CSV（ニューラルネットの予測）** を読み込む。  
   - ノート内では `submission.csv` や、作者が用意した **Kaggle データセット上のパス**（例：`vent-015a-pulp-fiction-inference/submission_better_than_median.csv`）が使われます。

2. **短縮パス（shortcut）**  
   - 事前にローカルで**全計算を回した結果**を別データセットとしてアップロードすると、**それを読み込んで**9時間超の計算を避けられる、という説明があります。

3. **`joblib.Parallel`** で**複数行を並列**に処理し、  
   - `submission_pi.csv`（最終的な提出）  
   - `pi_parameters.csv` / `p_parameters.csv`（推定した制御パラメータ）  
   などを保存します。

---

## 7. 初心者がハマりやすいポイント

| ポイント                      | 内容                                                                                                                     |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **単体では完結しない**             | 「**最初にNNなどで出した提出ファイル**」が必要。`ventilator_pressure_lightgbm.ipynb` のような「CSV だけで学習〜提出」の流れとは違う。                             |
| **実行時間**                  | フル計算は**約9時間以上**かかる、と書かれている。Kaggle ではキャッシュ用データセットを使う想定。                                                                 |
| **パス**                    | `../input/...` は **Kaggle のデータセット**前提。ローカルではフォルダ構成を合わせる必要がある。                                                          |
| **`mean_absolute_error`** | コード内で使われているが、**最初の import に `sklearn` が含まれていない**可能性がある。実行するなら `from sklearn.metrics import mean_absolute_error` を追加する。 |
|                           |                                                                                                                        |

---

## 8. まとめ（一言）

**`sample.ipynb` は、「ニューラルネットの提出結果を、P/PI 制御の逆算で上書きして精度を上げる」上級者向けの後処理ノート**です。  
**「コンペのデータから最初からモデルを学習する」**説明は、`ventilator_pressure_lightgbm.ipynb` の方が向いています。
