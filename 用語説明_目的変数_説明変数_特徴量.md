# 目的変数・説明変数・特徴量とは？（初心者向け）

機械学習や Kaggle でよく出てくる「目的変数」「説明変数」「特徴量」を、Ventilator Pressure Prediction を例にわかりやすく説明します。

---

## 一言でいうと

| 用語 | 一言でいうと |
|------|--------------|
| **目的変数** | 「**予測したいもの**」。モデルが当てにしたい答え。 |
| **説明変数** | 「**予測の材料になるもの**」。そのままモデルに入れる元のデータの列。 |
| **特徴量** | 「**モデルが実際に使う入力**」。説明変数から作った・選んだ、予測に使う変数のこと。 |

---

## 目的変数（ターゲット）

### 何か
**「これがいくつ（どれ）になるか当てたい」という列**のことです。  
英語では **target（ターゲット）** や **objective variable** といいます。

### このコンペでは
- **目的変数 = `pressure`（気道圧）**
- 「このタイミングでの気道圧はどれくらいか？」を予測するのがコンペの目的です。
- 学習データ（train）には `pressure` の正解が入っており、テストデータ（test）には入っていないので、モデルが予測して提出します。

### イメージ
「テストの点数を当てたい」とき、**目的変数 = テストの点数**です。

---

## 説明変数

### 何か
**「目的変数を予測するときに、手がかりとして使う列」**のことです。  
英語では **explanatory variable** や **predictor** といいます。

### このコンペでは（元データの列）
- `time_step` … そのステップの時刻
- `u_in` … 吸気バルブの開き具合（0〜100%）
- `u_out` … 呼気バルブが開いているか（0 or 1）
- （test には `pressure` はないので、説明変数には含めない）

これらが「このときの pressure はどれくらいか？」を決める**手がかり**です。

### イメージ
「テストの点数を当てたい」とき、**説明変数 = 勉強時間・授業出席率・前回の点数**など、「点数に効きそうな情報」です。

---

## 特徴量（feature）

### 何か
**「モデルに実際に渡す入力」**のことです。  
説明変数を**そのまま**使うこともあれば、**計算して新しく作った列**も含めて「特徴量」と呼びます。  
英語では **feature** といいます。

### 説明変数との違い（ここが大事）

- **説明変数** … 元データに**もともとある列**（time_step, u_in, u_out など）。
- **特徴量** … モデルが**本当に入力として使うもの**。  
  → 説明変数そのもの + **そこから作った新しい列**（ラグ、累積和、差分など）をまとめて「特徴量」ということが多いです。

つまり、  
**「説明変数から特徴量を作る」＝ 元の列だけでなく、ラグや累積など計算で作った列も加えて、モデルに入れる入力を用意すること**です。

### このコンペでは（ノートブックで使っている特徴量の例）
- そのまま使う: `time_step`, `u_in`, `u_out`
- 作って追加: `u_in_lag1`, `u_in_lag2`, `u_in_cumsum`, `u_in_diff`, `u_in_rolling_mean2`, `u_in_rolling_mean5`, `time_step_cumsum` など

これら**全部まとめて**「特徴量」と呼び、LightGBM にはこの特徴量だけを渡して `pressure` を予測します。

### イメージ
「テストの点数を当てたい」とき、
- 説明変数: 勉強時間、出席率、前回の点数
- 特徴量: 上記に加えて「勉強時間の先週比」「出席率×勉強時間」など**自分で計算して足した列**も含めた、「モデルに渡す入力ぜんぶ」

## 図で整理

```
【元データ（train）】
  id | breath_id | time_step | u_in | u_out | pressure
  ---+-----------+-----------+------+-------+----------
  1  | 0         | 0.0       | 0.0  | 0     | -1.89      ← 目的変数（予測したい）
  2  | 0         | 0.033     | 5.0  | 0     | 2.45       ← 目的変数
  ...

  説明変数（予測の材料）: time_step, u_in, u_out
  目的変数（予測したい）: pressure
```

```
【特徴量を作ったあと】
  元の説明変数 + 新しく作った列 = 特徴量（モデルへの入力）

  例: time_step, u_in, u_out,
      u_in_lag1, u_in_lag2, u_in_cumsum, u_in_diff,
      u_in_rolling_mean2, u_in_rolling_mean5, time_step_cumsum
      → これらをまとめて「特徴量」と呼ぶ
```

```
【モデル（LightGBM）の入出力】
  入力: 特徴量（上記の列）
  出力: pressure の予測値

  特徴量 ──→ [ LightGBM ] ──→ pressure の予測
```

---

## まとめ

| 用語 | 役割 | このコンペの例 |
|------|------|----------------|
| **目的変数** | 予測したい答え | `pressure`（気道圧） |
| **説明変数** | 元データの「予測の材料」になる列 | `time_step`, `u_in`, `u_out` |
| **特徴量** | モデルに渡す入力（説明変数 + 作った列） | 上記 3 つ + ラグ・累積・差分・rolling など |

- **目的変数** = 「当てたいもの」
- **説明変数** = 「もともとある手がかり」
- **特徴量** = 「説明変数から作った・選んだ、モデルが使う入力ぜんぶ」

ノートブックでは「特徴量を作る」セクションで、説明変数からラグや累積などを足して特徴量を作り、その特徴量で目的変数 `pressure` を予測しています。
