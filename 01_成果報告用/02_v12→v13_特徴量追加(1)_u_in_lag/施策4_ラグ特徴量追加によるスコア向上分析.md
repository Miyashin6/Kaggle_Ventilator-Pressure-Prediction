# 施策4: `u_in` ラグ特徴量追加によるスコア向上分析

## スコア変化サマリー

| バージョン | Private Score | Public Score |
|------------|--------------|-------------|
| V12（変更前） | 1.4893 | 1.4975 |
| V13（変更後） | 1.3690 | 1.3717 |
| **改善幅** | **−0.1203** | **−0.1258** |

> MAE（Mean Absolute Error）は小さいほど良い。Private / Public ともに約 0.12 の大幅改善。

---

## 実施した変更

`breath_id` ごとに `u_in` を 1〜4 ステップ後方にシフトしたラグ特徴量を追加。

```python
for lag in [1, 2, 3, 4]:
    df[f'u_in_lag{lag}'] = df.groupby('breath_id')['u_in'].shift(lag)
```

| 追加特徴量 | 意味 |
|------------|------|
| `u_in_lag1` | 1 タイムステップ前の吸気流量 |
| `u_in_lag2` | 2 タイムステップ前の吸気流量（従来の `u_in_lag` と同値） |
| `u_in_lag3` | 3 タイムステップ前の吸気流量 |
| `u_in_lag4` | 4 タイムステップ前の吸気流量 |

---

## なぜスコアが向上したのか

### 1. 人工呼吸器の気圧は「過去の吸気流量の履歴」で決まる

人工呼吸器が肺に送り込む気圧（`pressure`）は、瞬間の吸気流量（`u_in`）だけでなく、**直前数ステップにわたる流量の変化パターン**によって決定される。これは肺・呼吸回路の物理特性（コンプライアンス、抵抗）に起因する。

```
pressure[t] ≈ f(u_in[t], u_in[t-1], u_in[t-2], u_in[t-3], u_in[t-4], ...)
```

ラグ特徴量を加えることで、LightGBM が「現在の流量」だけでなく「流量の変化傾向・加速度」を木の分岐条件として使えるようになる。

---

### 2. ラグが与える情報の種類

| ラグの組み合わせ | モデルが推定できる情報 |
|-----------------|----------------------|
| `u_in[t]` のみ | 現在の吸気流量の大きさ |
| `u_in[t]` − `u_in[t-1]` | 流量の変化速度（一次微分に相当） |
| `u_in[t-1]` − `u_in[t-2]` | 変化の加速度（二次微分に相当） |
| `u_in[t-1]` 〜 `u_in[t-4]` の系列 | 吸気相の立ち上がり・ピーク・減衰パターンの形状 |

LightGBM の決定木は特徴量間の**組み合わせ（交互作用）**を自動的に学習するため、ラグ系列が揃うと「流量が急増中か」「ピーク直後か」などの文脈を木の深い分岐で捉えられる。

---

### 3. 1ステップラグ（`u_in_lag1`）の追加が特に効いた理由

変更前のモデルは `u_in_lag2`（2ステップ前）のみを持っていた。ここに `u_in_lag1` を加えると、モデルは「**直前1ステップでの流量変化**」を直接参照できる。

- 吸気相の立ち上がり初期（流量が急増するタイミング）は、lag1 と lag2 の差が大きく、pressure の変化も急峻
- この差をモデルが明示的に利用できるようになり、急変点での予測誤差が減少した

---

### 4. 4ステップまでカバーすることの意義

このデータの時間解像度（約 0.04 秒間隔）において、4ステップ = 約 0.16 秒分の履歴をカバーする。

- 人工呼吸器の1呼吸サイクルは通常 2〜4 秒
- 吸気相の立ち上がりは最初の 0.2〜0.5 秒に集中
- したがって **4ステップ（0.16 秒）の履歴は吸気相の形状を捉えるのに十分な窓幅**

5ステップ以上を加えた場合は、breath_id をまたぐ（前回の呼吸の情報が混入する）ケースが増えノイズになる可能性がある。

---

### 5. `u_in_lag2` だけでは不十分だった理由

| 状態 | `u_in_lag2` のみ | `u_in_lag1` 〜 `lag4` |
|------|-----------------|----------------------|
| 流量急増中 | 2ステップ前との差は分かるが1ステップ前の急変は見えない | lag1 − lag2 で急変を直接検出可能 |
| ピーク付近 | ピーク前後の対称性を捉えられない | lag1〜lag3 の並びでピーク形状を識別可能 |
| 緩やかな変化 | 変化が小さすぎて信号として弱い | 複数ラグの合算で緩やかな傾向も捕捉 |

---

### 6. LightGBM との相性

LightGBM は**線形回帰やニューラルネットと異なり、特徴量間の明示的な差分や比率を自分では作れない**。差分・比率などは特徴量として明示的に与える必要がある。ラグ特徴量を列として追加することは、実質的に「モデルに微分情報を渡す」ことに相当し、時系列の局所パターンを決定木で扱えるようにする標準的な手法。

---

## まとめ

| 観点 | 説明 |
|------|------|
| **物理的背景** | 気圧は流量の時間積分・変化率に依存するため、過去の流量値が直接的な予測因子 |
| **情報の追加** | lag1 の追加で「1ステップ前の変化量（一次微分）」をモデルが利用可能に |
| **系列幅の適切さ** | 4ステップ（≒0.16 秒）は吸気立ち上がりをカバーしつつ、呼吸間のノイズを避ける適切な窓幅 |
| **LightGBM との相性** | 決定木は時系列の差分を自動生成できないため、ラグ列の明示的な追加が精度改善に直結 |
| **改善幅** | Private −0.1203 / Public −0.1258 は、施策の中でも最大級の効果 |
