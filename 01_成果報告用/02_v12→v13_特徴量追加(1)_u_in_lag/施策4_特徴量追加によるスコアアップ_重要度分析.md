## 各特徴量が重要な理由（機械学習 × 呼吸器の物理）

---

### 1位: `u_in_cumsum`（56.4%）— 圧倒的に重要

```python
train["u_in_cumsum"] = train.groupby("breath_id")["u_in"].cumsum()
```

#### なぜ重要か

人工呼吸器の気道内圧（pressure）は、**吸気量の積算値に強く依存する**。

```
肺の弾性（コンプライアンス）の式:
    Pressure ≈ Volume / C + R × Flow

Volume（容量）≈ u_in の累積和 = u_in_cumsum
```

つまり `u_in_cumsum` は「今まで肺にどれだけ空気が入ったか」を表しており、
物理的に圧力と最も直結する値。モデルがこれを最も重視するのは**物理法則と一致した正しい学習**と言える。

#### gain vs split の差異

| 指標 | 値 | 順位 |
|------|----|------|
| Gain | 56.44% | **1位** |
| Split数 | 24,808 | **2位**（time_step の次） |

gain が圧倒的 1 位 = **1回の分岐で得る情報量が非常に大きい**。つまり u_in_cumsum 一本で粗いが大局的な圧力値をモデルが把握できている。

---

### 2位: `R_C_div`（R/C 比、10.8%）— 追加した中で最大の貢献

```python
train["R_C_div"] = train["R"] / train["C"]
```

#### なぜ重要か

呼吸器の設定値 R（抵抗）と C（コンプライアンス）の比は、**回路の時定数 τ = R × C に密接に関連**する。

```
時定数: τ = R × C   (breath が安定するまでの時間の目安)
R/C 比: R と C のバランス → 吸気波形の「立ち上がり特性」を表す
```

| R/C 比が大きい（例: R=50, C=10） | 抵抗が大きく、圧力上昇が急峻 |
| R/C 比が小さい（例: R=5, C=50） | 抵抗が小さく、圧力上昇が緩やか |

R と C を**個別に特徴量に持つだけでは木の2段分岐が必要**だが、R_C_div として与えると 1 段の分岐でこの関係を捉えられる。

#### `R_C_mul`（0.65%）との比較

`R_C_mul = R × C` は R_C_div より 16倍以上 gain が低い。

- `R × C`（時定数）は **breath 全体のスケール** を表すが、それは `u_in_cumsum` が既に捉えている
- `R / C` は **吸気の立ち上がり特性（急峻 or 緩やか）** という、累積和では捉えられない情報を追加している

→ **R_C_mul は削除候補、R_C_div は維持すべき特徴量**

---

### 3位: `u_in_diff2`（8.65%）— diff2 が diff1 より有効な理由

```python
train["u_in_diff2"] = train.groupby("breath_id")["u_in"].diff(2).fillna(0)
```

#### なぜ diff2 > diff1 なのか（Gain: 8.65% vs 1.35%）

| 特徴量 | 定義 | 意味 |
|--------|------|------|
| `u_in_diff1` | `u_in[t] - u_in[t-1]` | 直前 1 ステップの変化量 |
| `u_in_diff2` | `u_in[t] - u_in[t-2]` | 2 ステップ前からの変化量 |

- diff1 は **ノイズ（測定誤差や微細な揺れ）の影響を受けやすい**
- diff2 は 2 ステップ分の変化を見るため、**滑らかなトレンドを捉えやすい**
- また `u_in_lag`（= 2ステップ前の u_in）と組み合わせると、diff2 は u_in と u_in_lag の差分でもあり、**加速度的な変化**を表している

> `u_in_diff1` の split 数（18,294）は多いが gain が低い（1.35%）。
> これは「細かな補正に多用されているが、1回の分岐での情報量が少ない」状態 = **冗長特徴量の典型的な症状**。

---

### 5位: `u_in_lag`（6.16%）— 過去の吸気量が圧力予測に有効

```python
train["u_in_lag"] = train.groupby("breath_id")["u_in"].shift(2).fillna(0)
```

`u_in_cumsum` が「今まで全体の積算量」を表すのに対し、`u_in_lag` は「2ステップ前の瞬時値」を表す。

- `u_in_cumsum` が **大域的（長期的）な傾向** を捉える
- `u_in_lag` が **局所的（直近）な時系列変化** を補完する

→ 2 つが組み合わさることで「積算的に圧力がどこまで上がっているか」と「直近の吸気ペースはどうか」を同時にモデルが学習できる。

---

### 11位: `u_out`（0.0%）— 施策1 の正しさを裏付ける証拠

```
u_out の gain: 0
u_out の split 数: 0
```

**モデルは u_out を一切使わなかった。**

これは施策1（「u_out=1 の呼気行を訓練データから除外する」）の判断が**正しかったことを定量的に証明**している。

```
なぜ u_out が不要か:
  訓練データを u_out=0（吸気相）のみに絞っているため、
  訓練データ内の全行で u_out = 0（定数）。
  定数列は分岐に使えないため、重要度が自動的に 0 になる。
```

もし施策1を実施せず全データで学習していた場合、u_out は重要特徴量になっていたはずだが、
その場合 **u_out=1（呼気）の行の pressure も予測しようとしてしまい、精度が下がる**。

## 特徴量重要度から見るスコアアップの全体像

```
施策4 追加前（元特徴量のみ）:
  time_step, u_in, u_out, R, C, u_in_cumsum
  → モデルは u_in の「今の値」と「累積」しか知らない
  → 時系列の変化方向・速度・RCバランスは学習できない

施策4 追加後（lag + diff + RC比）:
  + u_in_lag   → 「2ステップ前と今」を比較できる
  + u_in_diff2 → 「2ステップ分の変化速度」がわかる
  + u_in_diff1 → 「1ステップの微細変化」（補助的）
  + R_C_div    → 「抵抗とコンプライアンスのバランス」がわかる
  + R_C_mul    → （ほぼ重複情報、効果薄）
```

| フェーズ | Public Score | 説明 |
|---------|-------------|------|
| V12（施策4前） | 1.4975 | 時系列変化が見えない |
| V13（lag追加） | 1.3717 | 過去との比較ができるように |

## 今後の施策への示唆

### 削除候補（重要度が低い特徴量）

| 特徴量 | Gain | 削除理由 |
|--------|------|---------|
| `R_C_mul` | 0.65% | `R_C_div` で代替可能、R×Cは時定数として u_in_cumsum に内包 |
| `u_in_diff1` | 1.35% | split 多・gain 低 = ノイズ的な分岐に使われている可能性 |

### 追加候補（重要度上位を踏まえた考察）

| 候補特徴量 | 根拠 |
|---------|------|
| `u_in_lag2`〜`u_in_lag4` | u_in_lag（lag2）が有効 → lag3/4 で長期記憶を強化 |
| `u_in_cumsum_normalized` | cumsum を breath 内の最大値で正規化 → スケール依存を除去 |
| `time_step_normalized` | breath内のどの位置（0〜1）にいるか → 位置エンコード的な役割 |
| `u_in_rolling_mean_3` | 直近3ステップの平均 → diff よりノイズに強い変化量 |

---

## まとめ

施策4で追加した特徴量が **全体の約 28%** の予測力を担い、スコアアップに直接貢献した。
特に重要だったのは：

1. **`R_C_div`（R/C 比）**: 追加特徴量の中で最大貢献（10.8%）。吸気の立ち上がり特性という物理的に意味ある情報を提供
2. **`u_in_diff2`（2ステップ差分）**: 8.65% の貢献。ノイズに強い変化速度情報を提供
3. **`u_in_lag`**: 6.16% の貢献。局所的な時系列変化を u_in_cumsum の大域情報と補完

一方 `u_out` の重要度が **0** であることは、施策1（吸気相のみ学習）の判断が**物理的にも統計的にも正しかった**ことの定量的な裏付けとなっている。
