# u_out=1 フィルタリングによる Public Score 向上の分析

**施策:** 学習データから `u_out=1`（呼気相）の行を除外し、`u_out=0`（吸気相）のみで学習
**結果:** Public Score（MAE）が **0.07 改善**

---

## 1. 前提知識：u_out とは何か

| 値 | 意味 | 呼吸フェーズ | 圧力の挙動 |
|----|------|-------------|-----------|
| `u_out = 0` | 呼気弁が**閉じている** | **吸気相（Inspiratory Phase）** | 空気が肺に送り込まれ圧力が上昇 |
| `u_out = 1` | 呼気弁が**開いている** | **呼気相（Expiratory Phase）** | 空気が外へ排出され圧力が急降下 |

1回の呼吸（breath_id 単位・80ステップ）は、吸気相と呼気相の2つの全く異なるフェーズで構成されている。

---

## 2. なぜ u_out=1 を除外するとスコアが上がるのか

### 2-1. 評価指標の定義（最大の理由）

コンペの公式評価は以下のとおり定義されている：

> **スコアは、各呼吸の「吸気相（inspiratory phase）」における予測圧力と実測圧力の MAE である。呼気相（expiratory phase）はスコアに含まれない。**

すなわち、テストデータ全行のうち **`u_out=1` の行は採点対象外** である。
MAE の計算は `u_out=0` の行のみで行われる。

```
MAE = (1/n) * Σ|予測pressure - 実測pressure|   ← u_out=0 の行のみ
```

これが最も根本的な理由であり、以降の理由はすべてこの事実から派生する。

---

### 2-2. 呼気相は「採点されない雑音」として学習に悪影響を与える

`u_out=1` の呼気相では、圧力（pressure）の挙動が吸気相と**物理的に全く異なる**。

| フェーズ | 圧力の動き | 支配的な物理プロセス |
|---------|-----------|-------------------|
| 吸気相（u_out=0） | ゆるやかに上昇、R・C・u_in に依存した複雑な曲線 | 気道抵抗・肺コンプライアンスに従った充填 |
| 呼気相（u_out=1） | 急激に降下、ほぼ一定の低い値に収束 | 弁開放による急速な圧力解放 |

**呼気相を学習データに含めると何が起きるか：**

```
学習データに u_out=0 と u_out=1 が混在
           ↓
モデルは「2つの全く異なる圧力パターン」を同時に学習しようとする
           ↓
吸気相の複雑なパターンにとって、呼気相のデータは「ノイズ」として機能
           ↓
吸気相の予測精度が低下
           ↓
採点対象（u_out=0）の MAE が悪化
```

---

### 2-3. クラス不均衡：呼気相が学習を支配しやすい

1回の呼吸（80ステップ）において、吸気相と呼気相の構成比はおおよそ以下の通り：

```
吸気相（u_out=0）：約 20〜30 ステップ
呼気相（u_out=1）：約 50〜60 ステップ
```

呼気相は吸気相より行数が多い（≈ 2〜3倍）ため、フィルタリングなしで学習すると：
- 損失関数（MAE）の大部分を「採点されない」呼気相が占める
- モデルが呼気相の低圧パターンに引っ張られ、吸気相の予測が歪む

→ フィルタリングにより、モデルが **採点される吸気相に100%の表現力を集中** できる。

---

### 2-4. 特徴量 u_out の情報冗長性の解消

フィルタリング後は全サンプルが `u_out=0` で均一になるため：

- `u_out` 列自体が**定数（= 0）**となり、特徴量として意味を持たなくなる
- モデルは `u_out` に依存せず、**吸気相の圧力予測に直接寄与する特徴量**（`u_in`、`R`、`C`、`time_step` など）に集中できる
- 特徴量空間が整理され、木の分岐が本質的な情報に使われる

---

### 2-5. 目的変数の分布が単純化される

フィルタリング前後の `pressure` の分布：

```
フィルタリング前（u_out 混在）：
  - 低圧（呼気相）と高圧（吸気相）の二峰性分布
  - 分布が広く、モデルが「平均的な圧力」に引きずられる

フィルタリング後（u_out=0 のみ）：
  - 吸気相の圧力分布のみ（単峰性に近い）
  - モデルが吸気相の圧力範囲に特化した予測ができる
```

目的変数の分布が単純化されることで、LightGBM の木の分割がより精密になる。

---

## 3. 整理：なぜ MAE が 0.07 改善したか

```
フィルタリングによる改善メカニズム（まとめ）

1. 評価仕様との整合（直接効果）
   └─ 採点される吸気相のみを学習 → 評価対象データで最適化できる

2. 学習データ品質の向上（間接効果）
   ├─ 呼気相ノイズの除去 → 吸気相パターンをより正確に学習
   ├─ 不均衡解消 → 採点対象に学習リソースを集中
   └─ 目的変数の分布単純化 → 精密な分割が可能に

3. 特徴量の有効活用
   └─ u_out が定数化 → 他の特徴量の重要度が正確に反映される
```

## 5. 機械学習の観点から見た位置付け

この施策は、機械学習における古典的な原則「**学習データと評価データの分布を一致させる**」の典型例である。

| 原則 | 今回の適用 |
|------|-----------|
| **Train-Test Distribution Match** | 評価対象（u_out=0）と同じ分布のデータで学習 |
| **Task-Specific Training** | タスクに不要なデータ（呼気相）を学習から除外 |
| **Label Noise 除去** | 別フェーズの圧力パターンを「ノイズ」として排除 |
| **Class Imbalance の解消** | 多数派（呼気相）による学習の歪みを防ぐ |
