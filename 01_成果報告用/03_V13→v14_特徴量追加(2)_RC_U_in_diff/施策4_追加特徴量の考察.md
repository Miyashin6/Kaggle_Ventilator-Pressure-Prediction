# 施策4 拡張: 追加特徴量の考察

## 前提：現在の特徴量セット（V13時点）

| 特徴量           | 種別      | 内容                    |
| ------------- | ------- | --------------------- |
| `time_step`   | 生       | breath内の経過時間          |
| `u_in`        | 生       | 吸気流量（コントローラ入力）        |
| `u_out`       | 生       | 呼気バルブ（0=吸気中, 1=呼気中）   |
| `R`           | 生（カテゴリ） | 気道抵抗（5, 20, 50）       |
| `C`           | 生（カテゴリ） | 肺コンプライアンス（10, 20, 50） |
| `u_in_cumsum` | 集計      | breath内のu_in累積和       |
| `u_in_lag1〜4` | 時系列     | 1〜4ステップ前のu_in         |
|               |         |                       |

> 現在のスコア（V13）: Private 1.3690 / Public 1.3717

## 追加特徴量の候補と考察

---

### A. `u_in` 差分特徴量（変化速度・加速度）

#### 意図

`u_in_lag1〜4` の追加により「過去の値そのもの」は与えられた。
しかし LightGBM の決定木は **列間の引き算を自動では行わない**。
差分を明示的に特徴量化することで、流量の「増加中か／減少中か／加速中か」をモデルが直接利用できる。

```python
# 一次差分（変化速度）
df['u_in_diff1'] = df.groupby('breath_id')['u_in'].diff(1)  # u_in[t] - u_in[t-1]
df['u_in_diff2'] = df.groupby('breath_id')['u_in'].diff(2)  # u_in[t] - u_in[t-2]

# 二次差分（加速度）
df['u_in_diff1_diff1'] = df.groupby('breath_id')['u_in_diff1'].diff(1)
```

#### 物理的背景

- 気圧 = 抵抗成分（R × 流量）＋弾性成分（1/C × 体積）
- 弾性成分 ∝ `u_in_cumsum`（体積 ≈ 流量の時間積分）
- **抵抗成分の変化** ∝ 流量の変化速度（`u_in_diff1`）

#### スコア向上見込み

| 項目 | 見込み |
|------|--------|
| 改善幅の目安 | −0.03〜−0.08 |
| 根拠 | lag特徴量（−0.12）の効果の一部は差分情報によるものと考えられる。差分を明示的に与えることで重複なく情報を補完できる |
| リスク | lag1〜4と高い線形相関を持つため、情報の重複が発生しやすい。gainベースの重要度で効果を確認してから取捨選択が必要 |

---

### B. `u_in` rolling統計量（局所的な分布特性）

#### 意図

ラグ・差分は「点」の情報。rolling統計量は「区間」の統計を与え、
吸気相のピーク付近・安定区間・減衰区間といった **フェーズの文脈** をモデルが捉えやすくなる。

```python
for w in [3, 5]:
    df[f'u_in_roll_mean_{w}'] = df.groupby('breath_id')['u_in'].transform(
        lambda x: x.rolling(w, min_periods=1).mean()
    )
    df[f'u_in_roll_std_{w}'] = df.groupby('breath_id')['u_in'].transform(
        lambda x: x.rolling(w, min_periods=1).std().fillna(0)
    )
    df[f'u_in_roll_max_{w}'] = df.groupby('breath_id')['u_in'].transform(
        lambda x: x.rolling(w, min_periods=1).max()
    )
```

#### 物理的背景

- 窓幅3（≈0.12秒）は吸気立ち上がりの「急峻さ」を表す
- 窓幅5（≈0.20秒）は吸気ピーク前後の「形状の滑らかさ」を表す
- `roll_std` が大きい区間 = 流量変動が激しい = 気圧も変動しやすい

#### スコア向上見込み

| 項目 | 見込み |
|------|--------|
| 改善幅の目安 | −0.02〜−0.06 |
| 根拠 | トップ解法のノートブックでも rolling mean/std は頻出の特徴量 |
| リスク | 窓幅の選択が効果に影響する。breath_id 境界での min_periods 処理を誤るとデータリークが発生 |

---

### C. `R × C` 交互作用特徴量

#### 意図

`R`（抵抗）と `C`（コンプライアンス）は **肺の物理特性を表すカテゴリ変数**。
両者を組み合わせた「RC時定数」は、肺が同じ流量入力に対してどれだけ速く気圧が上がるかを決定する。
現状は `R` と `C` を個別の特徴量として与えているが、交互作用を明示することで
モデルが「RC=5×10の組み合わせ」など特定パターンを学習しやすくなる。

```python
# カテゴリ結合
df['R_C'] = df['R'].astype(str) + '_' + df['C'].astype(str)  # label encoding / onehot

# 物理的なRC時定数（抵抗×コンプライアンス）
df['RC_product'] = df['R'] * df['C']      # 時定数 τ = R×C
df['R_div_C']    = df['R'] / df['C']      # 流量に対する気圧感度の比
```

#### 物理的背景

- RC 回路モデルで気道系を近似すると: `pressure ≈ R * u_in + (1/C) * ∫u_in dt`
- `R * C`（時定数）が大きいほど気圧の応答が遅い = ラグ特徴量の影響がより長く残る
- R=50, C=10の組み合わせ（高抵抗・低コンプライアンス）では気圧が急峻に変化する

#### スコア向上見込み

| 項目     | 見込み                                                                   |
| ------ | --------------------------------------------------------------------- |
| 改善幅の目安 | −0.05〜−0.15                                                           |
| 根拠     | R・C の組み合わせは3×3=9パターンしかなく、各パターンで圧力挙動が大きく異なる。モデルがこの違いを捉えられていれば大きな改善になるu |
| リスク    | LightGBM はカテゴリ変数の交互作用をある程度自動で学習できるため、改善幅は過大評価になる可能性もある                |

### D. breath内の位置・フェーズ情報

#### 意図

`time_step` は絶対時間だが、「breath全体の中での相対位置（何割進んだか）」や
「吸気フェーズの何ステップ目か」という情報はモデルにとってより直接的に有用。

```python
# breath内での相対位置（0〜1に正規化）
df['time_step_norm'] = df.groupby('breath_id')['time_step'].transform(
    lambda x: (x - x.min()) / (x.max() - x.min())
)

# breath内でのステップ番号（0始まり）
df['step_in_breath'] = df.groupby('breath_id').cumcount()

# 吸気フェーズ内でのステップ番号（u_out=0の区間のみカウント）
df['step_in_inspiration'] = df[df['u_out'] == 0].groupby('breath_id').cumcount()
df['step_in_inspiration'] = df['step_in_inspiration'].fillna(-1)  # 呼気中は-1

# breath全体の長さ（ステップ数）
df['breath_len'] = df.groupby('breath_id')['time_step'].transform('count')
```

#### 物理的背景

- 吸気フェーズの序盤（立ち上がり）・中盤（定常）・終盤（減衰）で圧力の変化パターンが異なる
- `time_step` の絶対値ではなく相対位置を与えることで、異なるサンプリング密度や息の長さへの汎化性が上がる

#### スコア向上見込み

| 項目     | 見込み                                                                                             |
| ------ | ----------------------------------------------------------------------------------------------- |
| 改善幅の目安 | −0.02〜−0.05                                                                                     |
| 根拠     | `time_step` は既に特徴量に含まれているため、正規化による改善幅は限定的。一方 `step_in_inspiration` は吸気フェーズ内の位置を明示するため新規情報を追加できる |
| リスク    | `time_step` との多重共線性が高く、重要度上は `time_step` に吸収される可能性がある                                           |

### E. breath単位の集約統計量（グローバル特徴量）

#### 意図

現在の特徴量はすべて「行（タイムステップ）単位」の情報。
breath全体の統計（ピーク流量・平均流量・流量の分散）を各行に付与することで、
「このbreathは全体的に強い吸気か／弱い吸気か」という文脈情報をモデルが使えるようになる。

```python
agg = train.groupby('breath_id')['u_in'].agg(
    u_in_breath_max='max',
    u_in_breath_mean='mean',
    u_in_breath_std='std',
    u_in_breath_sum='sum',
).reset_index()

train = train.merge(agg, on='breath_id', how='left')
```

#### 物理的背景

- breath全体のu_in積分値（≒ `u_in_breath_sum`）は「1回換気量」に対応し、目標気圧の水準を決定する
- `u_in_breath_max` が高いbreathは全体的に高い気圧レンジで推移する

> **注意:** テストデータに対してもこの集計が適用可能か確認が必要。テストデータの各breathが完全に存在する前提で処理する。

#### スコア向上見込み

| 項目 | 見込み |
|------|--------|
| 改善幅の目安 | −0.03〜−0.08 |
| 根拠 | breath全体の換気量は圧力スケールを決定する重要因子。現在の特徴量には含まれていない |
| リスク | テストデータでも同様の集計が必要であり、breath単位での情報が完全に利用可能であることを前提とする |

---

### F. `u_in_cumsum` の派生特徴量

#### 意図

現在の `u_in_cumsum`（累積和）は体積の近似値。ここから派生して
「累積和に対する現在の流量の比率」や「累積和の変化率」を与えると
モデルが「体積充填の進捗」を把握しやすくなる。

```python
# 累積和に対する現在流量の割合（吸気フェーズでの充填速度）
df['u_in_over_cumsum'] = df['u_in'] / (df['u_in_cumsum'] + 1e-6)

# u_in_cumsum のラグ（体積変化の履歴）
df['u_in_cumsum_lag1'] = df.groupby('breath_id')['u_in_cumsum'].shift(1).fillna(0)
```

#### スコア向上見込み

| 項目 | 見込み |
|------|--------|
| 改善幅の目安 | −0.01〜−0.03 |
| 根拠 | `u_in_cumsum` は既に含まれており、その派生は情報の追加量が小さい |
| リスク | `u_in` と `u_in_cumsum` から計算できる情報のため、モデルが既に近似的に学習している可能性が高い |

## 推奨実施順序

```
Step 1: R×C 交互作用（RC_product）の追加
        → 物理モデルとの一致度が高く、カテゴリ数が少ないため効果を確認しやすい

Step 2: u_in_diff1（一次差分）の追加
        → ラグ特徴量の補完として情報の重複が少ない

Step 3: breath単位集約統計量の追加
        → u_in_breath_max / u_in_breath_sum が圧力スケールに直結

Step 4: rolling統計量の追加
        → 窓幅のチューニングが必要なため、Step1〜3の効果確認後に実施
```

---

## まとめ

- V13時点での特徴量は「現在値」と「ラグ（過去の値）」が中心
- 最も追加情報量が大きいのは **R×C の交互作用**（肺の物理特性を直接表現）と **u_in差分**（流量変化速度の明示化）
- **breath単位の集約統計量**は現在の特徴量では表現できていない「1回換気量の規模感」を補完できる
- rolling統計量・位置情報は改善幅は小さいが、他特徴量との組み合わせで効いてくる可能性がある
